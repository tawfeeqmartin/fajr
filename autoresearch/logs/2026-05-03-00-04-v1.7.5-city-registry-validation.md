# AutoResearch Run — 2026-05-03 00:04 UTC — v1.7.5

## Hypothesis
Issue #47 surfaced four false positives during a 52-coord agiftoftime sample
worldwide: Cairo→Giza, Toronto→USA, KL→Shah Alam, Singapore→Johor Bahru.
The user's explicit ask was to **systematically validate the entire 375-city
registry**, not whack-a-mole patch the four. Reviewer C subsequently
catalogued 23 distinct bug classes across 308 deliberately-hard coords
(border cities, dispute zones, sub-national admin overlaps, Mexico's NW
border with USA, Saudi Arabia's NE border with Iran, etc.).

The fix has two prongs:
1. **Engine-level**: introduce a city/country sanity cross-check in
   `detectLocation` so a candidate city's claimed `countryISO` must agree
   with `detectCountry`'s verdict — eliminates ~389 cross-border bbox
   leaks at one stroke (Johor Bahru extending across the Causeway, etc.).
2. **Registry + detectCountry**: tighten or reorder country bboxes for the
   border-overlap cases that the cross-check exposes.

## Change made

### 1. `src/engine.js` — detectLocation refactor

Added an `ISO_TO_ENGINE_COUNTRY_LOCAL` mapping table (mirrors the build
script's table) so the city scan can map a city's `countryISO` to the
detectCountry vocabulary. Restructured `detectLocation` into a two-pass
city scan:

- **Pass A** (default): Skip a candidate city if its `countryISO` maps to
  a different engine-country than `detectCountry` returned. This is the
  cross-border bbox leak filter — it eliminates the Johor-Bahru-into-
  Singapore class of bug at one stroke.
- **Pass B** (fallback): If Pass A returned no match BUT a candidate's
  claimed country's bbox INDEPENDENTLY contains the coord (corroborated
  by a separate `COUNTRY_BBOX_TABLE` lookup), accept that candidate. This
  handles the Toronto / Montreal / Ottawa case where `detectCountry`
  returns USA first (USA's bbox extends north of the Great Lakes) but
  Canada's bbox also contains the coord — Toronto's claim is corroborated.

Added a city-aware country override: when a city is matched and its
`countryISO` maps to a known engine country, that country wins over
`detectCountry`'s verdict. This single-line override fixes Toronto
(country was 'USA', now 'Canada' because the city.countryISO=CA
agrees with Canada's COUNTRY_BBOX_TABLE entry).
